关于小飞熊

🐻 谁是小飞熊？

小飞熊，自从2001年陷入网络便不可自拔，2003年由原来的小熊更名为小飞熊，成为网络上第一头小飞熊；

大学毕业后一直从事互联网开发工作，经历过好多种Title，但还是最喜欢Software Engineer这个。

💡 防伪标记：为了防止与其他小飞熊混淆，我今后会在名字后附带上 lfbear 作为标记。

🛠️ 我的硬核武器库 (Tech Stack & Expertise)

  • 基础架构与云原生 (Cloud Native & Infra): Kubernetes / Docker / Service Mesh (Istio) / 自动弹性伸缩与大数据混部
  • 开发运维与工程化 (DevOps & CI/CD): GitLab CI / GitOps (ArgoCD) / Jenkins / 基础设施即代码 (IaC)
  • 私有算力折腾 (Compute Matrix): 深度学习多卡GPU工作站折腾 / TrueNAS / 软硬件物理装配调试 / 远程调优

🎨 小飞熊的爱好

  • 热衷于研究与折腾各种新奇的软硬件事物。
  • 构建高效的自动化工作流，追求极致的系统与开发体验。

🚀 这头熊都做过什么

因为当初从小熊更名为小飞熊的时候网络上面只有我一头小飞熊，但是现在看来有越来越多的小飞熊加入了这个队伍，所以还是在此梳理一下我都做了些什么，以防大家和其他熊混淆：

  • 🎓 中学时代：在校开办了一个叫做“小熊工作室”东东，组织志同道合的小伙伴写点程序，发点文章。编写并发布了数学运算我帮你, 物理公式计算, 文件加密我帮你II 等小程序；
  • 🎓 大学时代：编写了小飞熊下载系统, Bobtail-Bear 2006, ToyPHP，以及学校的就业信息网系统；
  • 💼 工作履历：
    • 某知名门户网站：早期从事互联网应用开发工作，后转做底层系统研发与技术管理。
    • 某互联网创业公司：前期做全栈开发助力业务快速成长。规模化以后转回后端，创建公司系统研发团队，在 DevOps，CI/CD，K8s 和 ServiceMesh 等领域玩耍的不亦乐乎。
    • 某互联网创业公司：主要负责千人级研发团队的云原生落地，涉及 Web 业务与 AI 大数据的混部、自动弹性、Service Mesh 等云原生技术的成功生产实践。
    • 海外某互联网创业公司：深度体验了海外的工作与生活，工作内容依旧是 DevOps 基础设施领域。
    • 目前：保密 🤫
  • 🎒 个人创业经历：在15年的创业浪潮下，分别尝试过 O2O 和 IoT 垂直行业的创业，而后退出。
